WHAT THE CASE STUDY REQUIRES
================================================================================

FlowAI is building an AI automation platform for healthcare. You must design an
end-to-end solution covering:

1. DATA INGESTION ARCHITECTURE
   - Ingest from 3 sources: EMR (clinical), RCM (billing), Referral Orders
   - Multi-source connectivity (APIs, batch files, webhooks)
   - Landing zone with medallion approach (Bronze → Silver → Gold)
   - Data quality controls (freshness, completeness, duplicates, integrity)
   - PHI encryption and HIPAA compliance

2. PATIENT 360 & ENTITY RESOLUTION
   - Resolve patient identities across EMR and RCM systems
   - Use deterministic matching (MRN, SSN-last4, phone, email)
   - Use probabilistic scoring (name similarity + DOB + address)
   - Build crosswalks from source IDs to canonical patient ID
   - Extend same approach to Provider/Counterparty graphs (law firms, PI firms)

3. ANALYTICS FRAMEWORK & BI-READY MODEL
   - Build star schema (dimensions + facts)
   - Create reports: counterparties by pending AR, patients by balance, denial rates
   - Support aging bucket analysis and AR tracking

EVALUATION CRITERIA:
  • Architecture correctness (lineage, replayability, schema evolution)
  • Data modeling quality (keys, normalization, extensibility)
  • Entity resolution rigor (confidence scoring, explainability)
  • Quality & governance (tests, monitoring, PHI controls)
  • Pragmatism (MVP path + scaling path)

RECOMMENDED FOR MVP:

   URL: https://synthetichealth.github.io/synthea/
   GitHub: https://github.com/synthetichealth/synthea

   Format: FHIR (R4/STU3), CSV, C-CDA

   URL: https://download.cms.gov/nppes/NPI_Files.html
   API: https://npiregistry.cms.hhs.gov/api-page
   What: 10-digit NPI provider identifiers (real, standardized)
   Format: CSV (comma-separated)
   Uses: Build dim_provider, identity_entity tables
   Cost: Free
   Setup: 5 minutes
   Update: Weekly/Monthly

3. SYNTHETIC CLAIMS DATA (You generate this)
   What: Map Synthea encounters to RCM claims
   Generates: rcm_claim_header, rcm_claim_line, rcm_remittance_835, AR snapshots
   Uses: Prototype RCM ingestion pipeline
   Cost: Free
   Setup: 1-2 hours (Python script)

4. SYNTHETIC REFERRAL ORDERS (You generate this)
   What: Generate referral orders with counterparty firms
   Generates: referral_order, referral_order_status_history
   Includes: Law firms, PI firms, clinics, employers as counterparties

   Setup: 1-2 hours (Python script)

KEY DECISIONS FOR YOUR SOLUTION
================================================================================

1. ENTITY RESOLUTION APPROACH
   Decision: Two-step (deterministic first, then probabilistic)
   Rationale: High-confidence matches first; probabilistic as fallback
   Confidence thresholds: 0.95+ deterministic, 0.75+ probabilistic

2. MEDALLION PATTERN
   Decision: Use Bronze → Silver → Gold layering
   Rationale: Clear separation of concerns; supports auditing & replay

3. DATA QUALITY OWNERSHIP
   Decision: Define SLAs per source system
   Rationale: Each source has different reliability expectations

4. PHI HANDLING
   Decision: Encrypt at rest (AES-256), in transit (TLS), audit all access
   Rationale: HIPAA required; build compliance from day one

5. MVP SCALE
   Decision: 1,000-5,000 synthetic patients initially
   Rationale: Proves concepts without infrastructure burden

6. IDENTITY GRAPH STORAGE
   Decision: Relational tables (identity_entity, identity_identifier, identity_edge)
   Rationale: Simpler than graph DB for MVP; scales to production warehouse
